# Token efficiency

1. Complete files, not partial snippets.
2. Skip basic React/TS explanations. Only teach unfamiliar patterns.
3. Batch related files in one response.
4. Comments for why, not what.
5. Architecture discussions: bullets, not essays.
6. Reference earlier explanations, don't repeat.
7. Show file tree before writing code.
8. No pleasantries — straight to the answer.
